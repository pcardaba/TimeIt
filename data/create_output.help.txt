create_output  -name output_name
	      [-specify (internal)|external]
              -launch_clock launch_clk
              -capture_clock capture_clk
              [-rclk_outputdly_max {outdly_expr}]
              [-rclk_outputdly_min {outdly_expr}]
	      [-fclk_outputdly_max {outdly_expr}]
              [-fclk_outputdly_min {outdly_expr}]
              [-rclk_oedly_max {oedly_expr}]
              [-rclk_oedly_min {oedly_expr}]
              [-fclk_oedly_max {oedly_expr}]
              [-fclk_oedly_min {oedly_expr}]
              [-rclk_latency_max {lat_expr}]
              [-rclk_latency_min {lat_expr}]
              [-fclk_latency_max {lat_expr}]
              [-fclk_latency_min {lat_expr}]
              [-data_edges {list_edges}]
	      [-hiz_edges {list_edges}]
	      [-high_edges {list_edges}]
	      [-low_edges {list_edges}]
	      [-unknown_edges {list_edges}]
              [-color (black)|green|red|blue|orange|purple]
              [-amplitude amp_value]
              [-lwidth line_width]
              [-visible]
              [-help]
  

  output_name   : Mandatory. The output signal name.
  specify     : Optional. Choice: (default is "internal")
                   internal   : Output delays are given referring to internal logic.
                                Usually post-layout extracted by using STA (Static Timing Analysis).
                   external   : Output delays are given by external requirements.		   	       
		                Usually pre-layout, same as SDC "set_output_delay" constraints.
  launch_clk     : Mandatory (unless -capture_clock is given). Name of the clock that launches the
                   data. The output signal is assumed to be generated at edge events of this clock.
  capture_clk    : Mandatory (unless -launch_clock is given). Name of the clock that captures the
                   data at the receiving end.
                   The launch and the capture clocks must be related: they must share the same
                   source clock (they may be the same clock, a source clock and a clock generated
                   from it, or two clocks generated from the same source clock). Their periods may
                   differ.
                   Giving only one of both means that the same clock launches and captures the data
                   (this is the former -refclock).
  outdly_expr    : Expression. Optional. Delay expression that shall resolved in a numeric value.
  		   The meaning of the values in -rclk/fclk_outputdly_max/min options depends on the
		   -specify option. Please beware of the differences:
                   When -specify is "external", if only rclk (rising clock) output delays are specified
 		   and not fclk (falling clock) output delays, then it is considered that capturing flip-flops
 		   are at rising egdes. Conversely if only fclk output delays are specified
 		   then it is assumed that capturing flip-flops are at falling edges.
 		   If both rclk and fclk delays are present then it is assumed that both
 		   edges are used for capture at the receiving flip-flops.
		   When -specify is "internal", rclk delays apply on lauching edge (as stated in edge lists)
		   If rising edge is in an edges lists and there is no rclk output delay specified then it
		   is assumed to be 0. Similarly if there is a falling edge in an edge lists and there is
		   no fclk output delay specified then it is assumed to be 0.
			       
	             -rclk_outputdly_max/min : max/min output delay when signal is launched by rising clock edge.
		     -fclk_outputdly_max/min : max/min output delay when signal is launched by falling clock edge.
		                              
					      
		     -rclk_latency_max/min: max/min rising clock edge sensitive flip-flop latency.
		     -fclk_latency_max/min: max/min falling clock edge sensitive flip-flop latency.

  oedly_expr     : Expression. Optional. Output enable delay expression that shall resolve in a numeric
                   value. Default is 0. Only relevant if "internal" -specify is used.
                   These options model the propagation delay through the output enable buffer, which
                   drives the output pad to or from Hi-Z state:
                     - From any data/high/low/unknown value transitioning to Hi-Z.
                     - From Hi-Z transitioning to any data/high/low/unknown value.
		     -rclk_oedly_max/min : max/min output enable delay when output enable is launched
		                            by a rising clock edge.
		     -fclk_oedly_max/min : max/min output enable delay when output enable is launched
		                            by a falling clock edge.

  lat_expr       : Expression. Optional. Clock latency expression that shall resolve in a numeric value.
                   Default is 0. Only relevant if "internal" -specify is used.
  list_edges     : List of reference clock edges. Edges are indexes numerically from 0 to n.
                   Simple number refers to any clock edge (rising or falling).
		   Compond NumerLetter referes to cycle and edge. "1P": Cycle 1 Posedge.
		   Important: Edges in lists always refer to lauching flip-flops.
		               ___    ___
		   Example: __/   \__/   \___/
		           0  1   2  3   4   5
			   0  1P  1N 2P  2N  3P

                   -data_edges : Contains the list of edges where data (multi-bit) is launched by the source
		                 at the specified edges.
	           -hiz_edges :  Contains the list of edges where signal turns to Hi-Z.
		   -high/low_edges : Edges where data/control signal (single-bit) goes high/low.
		   -unknown_edges : Edges where data value becomes unknown. 

  color          : Optional. Trace color. (black) | green| red | blue | orange | purple.
  amp_value      : Optional. Trace amplitude (integer value) in pixels. Amplitude shall be > 2.
                   Default is 40.
  line_width     : Optional. Trace line width in pixel (integer value). Default is 2.
  visible        : Optional. Trace is visible is -visible is given, hidden otherwise.
  num_cycles     : Optional. Show num_cycles. The number of clock cycles to be shown. 

Examples:


1.
   create_output -name "data_o<31:0>" \
                -specify external \
		-hiz_edges {0} \
		-data_edges {1P 2P 3P} \
		-unknown_edges {4P} \
		-rclk_output_dly_max {$out_delay} \
		-rclk_output_dly_min {$out_delay} 
		
Creates an output signal specified by using esternal delays. The signal will be represented
starting as Hi-Z (edge 0) and then at first clock rising (1P) data is launched. Data changes
at second and third rising edge (2P, 3P). At fourth rising clock rising edge the data becomes
unknown. Notice that only -rclk_... output delays are specified. Since -specify is "external"
this means that capture is assumed to be at rising edges. Remember edges in the lists refer to
launching flip-flops clock edges.
